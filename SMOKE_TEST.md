# 스모크 테스트 — balance_hj (부부 가계부)

**호스팅**: GitHub Pages (main 푸시 자동 배포)
**대상 변경**: 시트 z-index 동적 스태킹(`initBackGuard`의 `manageZ`) — 삭제/확인 오버레이가 다른 시트 위에 항상 표시되도록 trip_hj와 통일한 수정. 자세한 패턴은 `CLAUDE.md`의 "시트 z-index 동적 스태킹" 참고.

> 배포 반영 후 진행. main 푸시 → GitHub Actions 1~2분 + **강력 새로고침**(또는 PWA 재실행).
> 핵심 포인트: **삭제/확인 다이얼로그가 시트 위로 뜨고, 닫은 뒤 클릭 먹통·잔상이 없는가.**
> 참고: 이 앱엔 사진 기능이 없음 — 동일 코드 경로는 "확인 다이얼로그 스태킹"임.

## 체크리스트

- [ ] **내역 삭제 (핵심)**: 월간/연간 목록에서 항목 탭 → 액션시트 → `삭제` → 확인 다이얼로그가 **정상 표시**(액션시트 뒤에 안 가려짐) → 확인 → 삭제됨.
- [ ] **취소**: 삭제 확인에서 `취소` 또는 배경 탭 → 다이얼로그 닫히고 목록 정상.
- [ ] **다른 시트**: 필터/정렬 시트, 데이터 관리(구름) 시트, 앱 스위처, 날짜 선택기 각각 열고 → 배경 탭/스와이프/뒤로가기로 닫힘.
- [ ] **잔상 없음**: 위 시트들 닫은 뒤 화면 탭 → 먹통 없음.
- [ ] **헤더 고정(겸사)**: 월간↔연간↔기록 이동 시 좌상단 제목이 `가계부`로 **고정**(탭마다 안 바뀜).

## 합격 기준
- 삭제/확인 다이얼로그가 소스 시트 위에 보이고 동작 (가려짐 0건).
- 시트 닫은 뒤 클릭 먹통·잔상 0건.
- 배경 탭 / 아래 스와이프 / 뒤로가기 3가지 닫기 경로 모두 동작.
